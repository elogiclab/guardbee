Implementing the UserService

First of all you need to implement the UserService for local account management. When you use GuardBee your account model must implement the com.elogiclab.guardbee.core.User trait: the UserService will be a Play plugin that implements the com.elogiclab.guardbee.core.UserService abstract class having your model class as generic argument. The generic argument is for your facility, GuardBee framework only uses the User trait as argument.

Mandatory plugins

The framework needs three mandatory plugins to be activated:

  • the main GuardbeeService plugin. It must be the first started plugin.
  • the UserService implementation
  • the Authenticator(s)

Authenticators

Authenticators are responsable of the authentication process. Each authentication method needs an Authenticator to be implemented: LocalAccountAuthenticator is actually the only supported method (for now) and it is based on username/password mechanism by the UserService.obtainPassword method.

the User model does not have a password field. Depending on your security you can store the password directly in the User model or in a separate table/place/database.

GuardbeeService.PasswordProvider service is responsable of hashing/verifying password for LocalAccountAuthenticator. The default implementation uses Bcrypt hashing method, you are free to use any other method simply implementing the PasswordProvider abstract class and activating it as a normal plugin.

Stateful session

SessionManager is responsable of maintaining the stateful session. The default implementation writes the authentication informations (no passwords!) in a signed cookie; you can implement any other method, such as a server-side session management, by implementing the SessionManager abstract class and activating it as a normal plugin.

Protecting your URLs

To protect URLs add the SecuredController trait to your controller object and use the Authorized method passing your Action implementation. The first argument of Authorized is the authorization function: the authorization function is a simple function that receive the Authentication object and returns true if the user is authorized and false if he doesn't. There are several ready-to-use function in the authz package object, for more info please refer to the API documentation.

The Action function receive both request and Authentication object as function parameters. A controller function that does not need to be authorized can call the SecuredController.getAuthentication method to retrieve the current Authentication object. You can also call the GuardbeeService.getAutentication to retrieve the Authentication object from any part of code (but in any case you must have the request available).
